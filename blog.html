Let is block-scoped. It does not allow to redeclare variables. Also hoisting does not occur in Let. On the other hand Var is function-scoped. It allows to redeclare variables. And hoisting occurs in Var.

In a regular function its internal this value is dynamic. It depends on how function is invoked. But in arrow function there are no this, if we access this in the arrow function it will return this of the closest non-arrow parent function. we can use the new keyword on the regular function to create a new object. But arrow function cannot be called with new. In regular functions sometimes duplicate named parameters are allowed. While arrow function does not allow it.

Template strings are a powerful feature of modern JS released in ES6.It lets us insert/interpolate variables and expressions into strings without needing to concatenate like in older versions of JavaScript. It allows us to create strings that are complex and contain dynamic elements.Template literals are literals delimited with backtick (`) characters, allowing for multi-line strings, for string interpolation with embedded expressions, and for special constructs called tagged templates.
